# Workflow 2: Modify Code

Sau khi task đã được confirm:

1. **Lập Implementation Plan:** Nêu rõ mục tiêu, danh sách file cần sửa, cách sửa và rủi ro.
2. **Review Plan:** Nhận feedback/approval từ Human cho plan.
3. **Thực thi (Implementation):**
   - Code đúng convention.
   - Sửa gọn gàng trong scope đã duyệt.
   - Không chạm vào code đang chạy ổn định (stable code) nếu không thuộc scope.
   - Tuân thủ security rules và data leakage rules.
4. **Document Changes:** Ghi lại (log) những thay đổi, lý do thay đổi nếu cần thiết.
