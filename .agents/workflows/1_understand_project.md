# Workflow 1: Understand Project

Trước khi bắt đầu bất kỳ chỉnh sửa nào:

1. **Đọc yêu cầu (Business Requirement):** Trả lời 3 câu hỏi: Đang làm gì? Cho ai? Để đạt mục tiêu gì?
2. **Đọc tài liệu (Documentation):** Kiểm tra `working_rule.md`, `.agents/README.md`, và các file doc liên quan.
3. **Phân tích bối cảnh (Context Analysis):** Kiểm tra folder structure, data sources, logic hiện tại liên quan đến task.
4. **Summary & Confirm:** Tổng hợp cách hiểu, đưa ra các assumption, scope của task và xin ý kiến (confirm) từ Human.
5. **DỪNG LẠI:** Không viết code cho đến khi Human xác nhận summary.
